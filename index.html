🗺️ 전체 사용 흐름
1
⚙️ 공대 설정
공대 이름, 판매자 인센 %, 손님구인 인센 금액을 입력하고 멤버를 추가합니다. 패널티 항목과 용병/알바 지급 방식도 여기서 설정해요.
2
⚔️ 원정대 입력
날짜, 이번 회차 참여 멤버 체크 → 드랍템/손님지불금/공대지출 입력 → 패널티 지정 → 이 회차 저장 클릭!
3
💰 분배 결과
멤버별 최종 분배금 확인. 정산 완료된 멤버는 토글로 체크하세요. 상단 드롭다운으로 이전 회차 결과도 조회 가능합니다.
4
📊 통계 & 히스토리
전체 누적 정산금, 멤버별 수령액 통계, 회차별 히스토리를 확인할 수 있어요.
💾 저장 방식 안내
✅ 이 회차 저장 — 입력한 회차 데이터를 이 브라우저 내부(localStorage)에 저장합니다.
같은 컴퓨터, 같은 브라우저에서만 유지되며 브라우저 데이터 초기화 시 삭제될 수 있어요.
📥 백업 저장 — 모든 데이터를 JSON 파일로 다운로드합니다.
다른 기기에서 사용하거나 데이터를 보관하고 싶을 때 꼭 백업하세요!
📤 백업 불러오기 — 저장한 JSON 파일을 불러와 데이터를 복원합니다.
⚠️ 업데이트 시 새 파일을 드려야 합니다.
업데이트 파일을 받으면 백업 저장 → 새 파일에서 백업 불러오기 순서로 데이터를 이전하세요.
업데이트 문의 및 친추: 인게임 fourcard 로 친구 추가 꼭 부탁드립니다! 🙏
📚 용어 설명
용병 공대에 고용된 멤버. 분배 대상이지만 고정금액 또는 정규멤버 기준 % 로 별도 지급 설정 가능.
알바 용병과 동일한 구조. 구분이 필요할 때 사용. 지급 방식 별도 설정 가능.
손님 아이템(예: 혼테일 목걸이)을 먹으러 오는 외부인. 분배 대상이 아니며, 지불한 금액이 분배 풀에 합산됨.
손님구인 손님을 모집해온 멤버. 손님 1명당 설정된 인센 금액을 추가로 수령.
판매자 드랍템 판매를 담당하는 멤버. 판매 순수익의 설정된 % 를 인센으로 수령.
공대지출 알버프, 영혼의메아리 등 공대가 함께 사용한 비용. 분배 풀에서 먼저 차감됨.
패널티 사망, 노쇼 등 제재 금액. 고정금액 또는 기본 분배금의 % 차감. 차감액은 패널티 없는 멤버에게 균등 분배.
🧮 계산 방식 요약
① 분배 풀 계산
드랍 순수익 + 손님 지불 실수령 − 판매자 인센 − 손님구인 인센 − 용병/알바 고정지급 − 공대지출

② 기본 1인당 = 분배 풀 ÷ 참여 인원수

③ 패널티 적용
패널티 있는 멤버: 기본금 − 패널티 (금액 + 기본금×%)
패널티 없는 멤버: 기본금 + (패널티 총액 ÷ 무패널티 인원)

④ 추가 인센 적용
판매자 인센, 손님구인 인센, 용병/알바 % 지급 추가

⑤ 손님 지불금
손님이 낸 금액도 드랍템처럼 수수료 차감 후 실수령액이 분배 풀에 합산됩니다.
❓ 자주 묻는 질문
Q. 다른 컴퓨터에서 쓰고 싶어요.
A. 백업 저장으로 JSON 파일 받아서 옮기세요.

Q. 파일을 새로 받으면 데이터가 사라지나요?
A. 네! 반드시 백업 저장 후 새 파일에서 불러오기 하세요.

Q. 용병이랑 알바 차이가 뭔가요?
A. 기능은 동일해요. 구분이 필요할 때 라벨만 다르게 씁니다.

Q. 업데이트/버그 문의는요?
A. 인게임 fourcard 로 친추 후 문의해주세요! 🙏
